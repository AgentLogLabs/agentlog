# 🏗️ AgentLog Phase 1: 完整技术落地方案（Use Case 驱动版）

**核心定位**：AI 时代分布式工作流枢纽 (SSoT) **本期目标**：打通人机上下文，确立 Trace/Span 模型，通过“网关+旁路探针”双流采集机制，彻底解决“断点流转”与 Git 割裂问题。

---

## 🎯 核心业务场景 (Use Cases)

研发团队在执行架构重构时，必须确保以下两个核心 Use Case 能够完美跑通：

### Use Case 1: 人机混合微操接管 (Handoff Tracing)

- **痛点**：Agent 陷入死循环报错，CEO 强行中断，用 VS Code 手动修改代码修复 Bug。随后重新唤醒 Agent，但 Agent 记忆断层，不知道发生了什么，甚至覆盖了 CEO 刚写的代码。
    
- **预期流转**：
    

1. CEO 在 VS Code (配合 Cline) 修复 Bug，执行 `git commit -m "fix: 修复死循环"`。
    
2. **AgentLog Git Hook** 瞬间捕获该 Commit 的 Diff。
    
3. 网关将该 Diff 封装为 `Span (actor: human)`，并挂载到当前中断的 `TraceID` 下。
    
4. 重新唤醒 Agent 时，Agent 自动拉取该 TraceID 的最新上下文，立刻“看到”人类的修改，并顺着修改继续执行。
    

### Use Case 2: 跨 Agent 的“急诊交接” (JIT Context Hydration)

- **痛点**：`Builder Agent` 环境配置失败，只能在 Prompt 里简短总结一句“我失败了，报错是 XXX”，然后转交给 `Reviewer Agent`。`Reviewer` 因为拿不到完整的上下文和之前的错误尝试，只能重复踩坑。
    
- **期流转**：
    

1. `Builder` 放弃时，仅需传递：`"任务失败，请接手。TraceID: T-888"`。
    
2. `Reviewer` 收到指令后，主动调用 AgentLog 提供的 MCP 工具 `get_failed_attempts(trace_id="T-888")`。
    
3. 网关瞬间返回结构化的历史报错栈、输入参数及环境状态。`Reviewer` 零信息损耗接手修复。

## ⚙️ 核心技术架构：Gateway-Hub + Sidecar 模式

为支撑上述 Use Case，系统架构必须全面升级：

### 1. 数据模型重构 (ULID-based Trace & Span)

完全废弃现有的扁平 Session 结构，转向支持长周期、分布式追踪的树状结构。

- **Traces 表**：宏观任务流（对应 Use Case 中的一次完整研发流转）。
    
- **Spans 表**：具体执行动作。
    
- 字段：`id` (ULID), `trace_id`, `parent_span_id`, `actor_type` (human/agent), `payload` (JSON)。
    
- _说明_：Use Case 1 中人类的 Git Diff 和 Use Case 2 中 Agent 的报错日志，都作为不同的 Span 挂载在同一棵 Trace 树上。
    

### 2. 双流采集引擎 (Dual-Stream Ingestion)

外部工具（VSCode）和内部网络（OpenClaw）的日志必须汇聚。

**A. IDE 外部流（支撑 Use Case 1）**

- **网关升级**：Fastify 后端增加 `SSE (Server-Sent Events)` 路由，或通过 SSH StdIO 隧道，支持外部 IDE 稳定接入。
    
- **Git 拦截器**：编写标准化 `post-commit` Hook 脚本，负责拦截代码变更并推送到网关。
    

**B. OpenClaw 内部流（支撑 Use Case 2）**

- **旁路探针 (Telemetry Probe)**：严禁修改业务 Agent 逻辑。开发一个 OpenClaw 插件，挂载到原生 Hook (`on_agent_message`, `on_tool_execute`, `on_error`)。
    
- **无阻塞上报**：探针截获数据后，通过轻量级 HTTP REST API (`POST /api/spans`) 异步投递给 Gateway，确保不拖慢 Agent 运行速度。
    

### 3. 全局 TraceID 透传机制 (Trace Propagation)

- 人类在 VSCode 发起指令时生成 `TraceID`。
    
- 唤起 OpenClaw CLI 时通过 `--trace-id` 注入环境变量。
    
- 所有旁路探针强制读取该环境变量打标，完成“血脉继承”。
