发现当前记录比较简略——每个 span 只记录了工具名称，没有捕获具体参数和返回值：
{
"actorName": "exec",
"payload": {
"event": "tool",
"content": "{\"tool\":\"exec\"}",
"toolName": "exec", // ← 只有工具名
"durationMs": 0
}
}

后台日志也显示 hasInput: false，说明 beforeToolCall 钩子没有拿到工具的输入参数。

原因：当前 openclaw-agent-log 的 afterToolCall 钩子只传了工具名到 MCP，没有传完整的 args（输入参数）和 result（执行结果）。

要修复的话需要：在 afterToolCall 里把完整的 toolInput 和 toolOutput 也一起写入 span 的 payload。这需要改 openclaw-agent-log/src/index.ts 里的 createSpan 调用，把 args 和 result 也传进去。
